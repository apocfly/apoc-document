# 行为准则

## 前言

本行为准则（以下简称“准则”）是APOC天启模拟飞行连飞平台（以下简称“平台”）运行与管理的基础性规范文件。  
准则旨在明确平台全体成员——包括飞行员、管制员及其他形式登录者——在连飞过程中应遵守的基本行为标准，以保障飞行秩序、提升模拟飞行体验、促进社区良性发展。  
平台要求每一位成员认真阅读、理解并严格遵守本准则。所有成员在接入平台网络之时，即视为已知晓并同意接受本准则的全部条款。  
平台管理组保留对本准则的解释权与修订权，任何更新将通过官方渠道发布并即时生效。  
安全、尊重、专业与公序良俗是平台运行的核心原则。管理组将依据本准则对违规行为采取相应措施，包括但不限于警告、封禁账号及移出社区交流渠道等系列操作。  
成员之间应互为尊重，主动维护良好的连飞环境，共同推动平台的持续进步。

## 第一章 术语定义

### 1.1 机师/飞行员(Pilot)

指使用平台官方认可的客户端软件(`Swift`、`APOC Client`)连接至网络，并操纵航空器的人员。

### 1.2 管制员(Controller)

指使用平台指定的管制客户端(`Euroscope`)连接至网络，并为飞行员提供空中交通管制服务的人员。

### 1.3 向下兼容(Topdown)

一种管制运行模式，以实现对多机场的管制覆盖。在本平台，若管制员未在ATC INFO中明确说明不启用Topdown，则默认使用本运行模式。

### 1.4 学员(Under Mentoring)

指正处于管制学习阶段的人员。

### 1.5 教员(Instructor)

指负责教授学员并具备考核权限的人员。

### 1.6 教师(Mentor)

指负责教授学员但不具备考核权限的人员。

### 1.7 客座(Transfer and Visiting Controller)

指其他模拟飞行连飞平台的在职管制员，通过特定程序加入本平台并提供管制服务的方式。具体适用条件以本准则相关条款为准。

### 1.8 平移(Translation)

指一种使其他平台管制员更便捷加入本方式，由本平台负责其后续权限晋升的培训工作。本平台暂不接受此运行模式。

### 1.9 成员/会员(Member)

指已注册并持有本平台有效账户、可登录网络的人员。

### 1.10 网络(Network)

指本平台经官方认证、具有明确标识且供成员连接的连飞服务器及相关通信基础设施。

### 1.11 观察员（Observer）

指以非管制员身份连接至网络，仅用于观察或学习管制操作的人员，不具备管制能力。截止目前，平台仅对管制学员及以上开放OBS权限。

### 1.12 监察员（Supervisor）

指网络监察员，负责定期网络巡检，在遇有违反本准则的行为时进行处理。

### 1.13 热区（Hot Spot）

指跑道、滑行道等可能导致交通冲突的地区。

### 1.14 紧急情况（Emergency）

平台允许7600与7700应答机代码作为紧急情况使用，但禁止使用7500代码。

### 1.15 官方活动（Activity）

指由平台定期发布举办、保证管制覆盖的大型连飞活动。

### 1.16 团飞活动（Event）

指由玩家自发举办、无管制覆盖保证的小型连飞活动。

### 1.17 管制员信息（ATC INFO）

管制员在连线时填写的备注信息，可能包含下列内容：

1. 开设席位呼号；
2. 开设席位运行规则；
3. CPDLC/DCL登录代码、网络及运行规则；
4. 工单提交地址；
5. 其他补充信息。

## 第二章 总则

### 2.1 尊重每一个人

所有成员应彼此尊重，不得有不文明、不尊重或挑衅性的言行。

### 2.2 遵守国家法律法规

成员应遵守国家法律法规及本准则，避免过度玩笑或触犯他人底线，共同维护和谐的社区环境。

### 2.3 适用基本常识

阅读和使用本手册里各种规则时，请记住，在本手册中，基本常识永远适用。

### 2.4 人人平等

在网络中，每个人都是平等的，任何成员不得试图使用任何手段躲避原有的限制、惩罚等。

### 2.5 每人一个

每位成员最多只能持有一个账号，不可以注册多个账号，且不得使用他人账号登录，即使对方允许。

### 2.6 不得租借账号

成员持有的账号具有唯一性，不得进行租借。

### 2.7 不得多开

任何上线的会员仅允许在同一时间段，使用1个作为机组或管制的链接

下方的行为是不允许的：

1. 一个管制员链接与一个机组链接；
2. 两个管制员链接；
3. 两个机组链接。

特别注意：一个OBS链接与一个机组链接是允许的，但需要注意连线顺序；链接非OBS链接->连接语音软件->连接OBS链接

### 2.8 不得挂机

任何成员连接至网络不得在有管制空域中进行挂机。管制员不得在网络中进行挂机。

### 2.9 合理的名称

在连接到网络时，需要填写RealName，可以接受的值如下：

1. 注册CID
2. 真实英文姓名，比如：Junyu Yan；
3. 真实姓名缩写，比如：J.Y.

严禁填写默认RealName或不填写。

### 2.10 不得冒充

任何成员不得冒充网络上任何成员，如：网络上的监察员、亦或者是管制等。

### 2.11 倡导使用语音

平台鼓励以语音为首要通讯方式，同时保留文字通讯作为辅助。管制员应同时提供语音与文字服务，按需使用，二者不存在等级差异。

### 2.12 遵循公职人员的要求

所有成员连接至网络后，须服从公职人员（如监察员、管理组成员）的合法指令。若公职人员指令与管制员指令冲突，优先服从公职人员指令。拒不服从者将被强制断开网络连接。

### 2.13 先到先得

每位成员在连接至网络后，采用“先到先得”的方式对有限的资源进行分配，譬如：一个管制席位、一个机位等。

### 2.14 管制频率内禁止闲聊

管制频率中严禁出现与管制工作无关的内容。

## 第三章 飞行员行为准则

### 3.1 确保飞行能力

平台鼓励新手飞行员参与连飞活动，但若管制员认为当前空域繁忙，且该飞行员无法掌握航空器的基本操作，有权要求该飞行员暂时断开网络连接。

### 3.2 听从管制员指挥

飞行员一经连线，即视为愿意接受管制服务。凡不听从或不接受管制员合理指令者，管制员可以要求飞行员下线或不予提供管制服务。

### 3.3 共同承担责任

平台允许双人机组（即多位飞行员以同一身份连接至服务器）。若在执行航班过程中发生违规行为，所有参与该机组的成员共同承担相应处罚。

PF正常连线，PM飞行员以OBS连线，PM呼号为PF呼号加一个大写英文字母。

例如，PF以呼号CES2352连线，PM以呼号CES2352A连线。

### 3.4 使用真实天气

平台推荐所有飞行员使用实时真实天气，管制员将基于此提供管制服务。

### 3.5 及时更新导航数据

平台推荐所有飞行员使用最新的导航数据，管制员将基于此提供管制服务。

### 3.6 禁止在热区连线

热区（定义见[1.13](#1-13-热区-hot-spot)）包括跑道、滑行道等可能导致交通冲突的区域。飞行员不得在热区内进行初始连线。如遇特殊情况（如空中断线后重连），应确保不干扰其他航空器的正常运行。

### 3.7 提交飞行计划

飞行员在连线时应提交完整的飞行计划，以便其他机组及管制员查阅。

### 3.8 守听咨询频率

在暂无管制员上线的情况下，飞行员应守听122.800 MHz频率。

### 3.9 禁止倍速与暂停

未经管制员明确许可，飞行员不得在模拟器中开启倍速或暂停飞行。

### 3.10 禁止回放

任何成员在连接至服务器期间，不得观看回放（Replay），以避免影响空域秩序。

### 3.11 应答机代码限制

平台允许飞行员在必要情况下使用7700与7600应答机代码，但禁止使用7500应答机代码。

### 3.12 服从特情终止指令

管制员有权在必要情况下要求飞行员立即终止正在模拟的特情（Emergency），飞行员须立刻执行。

### 3.13 合理的学习

平台允许新手飞行员在合理范围内犯错误并从中学习，但禁止任何飞行员故意或长时间占用管制频率资源，影响正常通讯秩序。管理组及管制员有权对上述行为进行干预。

## 第四章 管制员行为准则

### 4.1 准时参与

教员与学员应准时参加约定的教学活动。如因故无法按时参加，须提前告知对方。

### 4.2 提前准备

不得在面试、考核等活动开始后才临时查阅理论资料。管制员应提前做好充分准备。

### 4.3 先观摩后上岗

新进管制员在首次独立上线前，须先进行必要的观摩或学习，熟悉本平台的运行规定及程序。

### 4.4 观察员呼号格式

以观察员（OBS）身份连接时，允许的呼号格式如下：

1. 使用“真实姓名缩写_OBS”格式，例如“HC_OBS”；
2. 使用"CID_OBS"格式，例如"2352_OBS"

公职人员执行公务时，可使用“真实姓名缩写_SUP”等表明身份的格式。

### 4.5 正确使用席位呼号

管制员仅允许以“呼号_席位代号”格式连接，且仅限以下席位代号：

| 席位代号      | 职责        |
|-----------|-----------|
| DEL       | 放行        |
| A_GND/RMP | 机坪（不负责放行） |
| GND       | 地面        |
| TWR       | 塔台        |
| APP       | 进近        |
| DEP       | 离场        |
| CTR       | 区域        |
| FSS       | 飞行服务站     |

### 4.6 视程距离限制

不同席位的最大视程距离规定如下：

| 席位              | 视程范围   |
|-----------------|--------|
| OBS             | 300海里  |
| DEL / GND / RMP | 20海里   |
| TWR             | 50海里   |
| APP / DEP       | 150海里  |
| CTR             | 600海里  |
| FSS             | 1500海里 |

### 4.7 先到先得与席位交接

管制席位遵循“先到先得”原则。若上一席位管制员在约定时段结束后或超出合理时长仍不主动交接或断开连接，须在收到提醒后5分钟内完成交接并退出，否则管理组有权强制断开。

### 4.8 遵守规则与程序

任何以管制员身份登录的成员，必须严格遵守本准则及平台发布的所有管制程序。

### 4.9 客座管制员不负责后续培训

以客座管制员身份加入本平台的其他平台在职管制员，其后续管制培训不由本平台负责。

### 4.10 客座管制员来源平台

仅允许以下模拟飞行平台的在职管制员通过客座管制员的身份加入本平台：

1. Vatprc（其他Vatsim分部的管制视情况客座）
2. Skyline
3. Xflysim
4. Chinaflier
5. Sino

### 4.11 遵守ATC INFO规定

管制员须严格遵守“管制员信息”（ATC INFO，定义见 [1.17](#1-17-管制员信息-atc-info) 中所载明的各项要求。在不同训练阶段（如学员、单人上岗等），应遵循对应的标注说明。

### 4.12 活动席位取消规则

管制员报名官方活动的管制席位后，如因个人原因需要取消，须提前至少24小时取消报名。若无法提前24小时取消，须自行联系顶班管制员，或征得上级席位（如塔台的与进近协调）同意后方可取消。

### 4.13 真实姓名填写

在连接到网络时，需要填写RealName，可以接受的值如下：

1. 真实英文姓名，比如：Junyu Yan；
2. 真实姓名缩写，比如：J.Y.

严禁填写默认RealName或不填写。

## 第五章 处罚规则

### 5.1 处罚种类

平台对违反本准则的行为，可视情节轻重及影响程度，单独或合并适用以下处罚措施：

1. 口头警告；
2. 书面警告；
3. 临时禁言（在官方交流群或网络通讯频道中）；
4. 临时封禁账号（限制登录网络）；
5. 永久封禁账号；
6. 移出官方交流群；
7. 列入黑名单（禁止再次注册或加入）。

### 5.2 一般违规行为及处罚

以下行为属于一般违规。一经查实，按情节予以处罚：

1. 情节轻微者：给予口头或书面警告或封禁1至24小时；
2. 情节较轻者：封禁1至7天；
3. 情节较重或屡教不改者：封禁7至30天；
4. 情节特别严重或对平台造成严重影响者：永久封禁。

一般违规行为包括：

1. 在管制频率中闲聊（违反2.14）；
2. 在有管制空域中挂机（违反2.8）；
3. 使用不合规的Realname（违反2.9）；
4. 观察员未按格式使用呼号（违反4.4）；
5. 未提交飞行计划（违反3.7）；
6. 作为管制员未遵守视程距离规定（违反4.6）；
7. 未按时取消活动席位且未安排顶班（违反4.12）；
8. 发表不尊重他人的言论（违反2.1）。

### 5.3 严重违规行为及处罚

以下行为属于严重违规。一经查实，按情节予以处罚：

1. 情节轻微者：给予口头或书面警告或封禁1至7天；
2. 情节较轻者：封禁14至30天；
3. 情节较重或屡教不改者：封禁30至60天；
4. 情节特别严重或对平台造成严重影响者：永久封禁。

严重违规行为包括：

1. 冒充他人（违反2.10）；
2. 持有多个账号或租借账号（违反2.4、2.5）；
3. 多开连接（违反2.7）；
4. 在热区连线并造成交通冲突（违反3.6）；
5. 未经许可倍速、暂停或回放（违反3.9、3.10）；
6. 使用7500应答机代码（违反3.11）；
7. 不服从管制员或公职人员指令（违反2.12、3.2）；
8. 屡教不改且被要求断开连接后拒绝执行（违反3.14）；
9. 管制员霸占席位拒不交接（违反4.7）；
10. 管制员使用不合规席位呼号（违反4.5）；
11. 不遵守或不尊重国家法律法规（违反2.2）；
12. 宣传反战等言论。

### 5.4 紧急情况特殊规定

如违规行为涉及7600或7700应答机代码的滥用（例如无真实特情情况下随意使用），按严重违规处理。

### 5.5 累犯加重规则

任何成员在一年内累计受到三次（含）以上书面警告或临时封禁的，后续违规按以下标准加重处罚：

1. 第4至6次：封禁1天、2天、3天（依次递增）；
2. 第7至9次：封禁1周、2周、3周（依次递增）；
3. 第10至12次：封禁1月、2月、3月（依次递增）；
4. 超过12次：永久封禁。

### 5.6 处罚程序

1. 公职人员（监察员、管理组成员）有权依据本准则当场采取必要措施（如禁言、断开连接）。
2. 临时封禁超过7天或永久封禁的处罚，须经管理组集体讨论决定。
3. 所有处罚决定应通过官方渠道（如系统通知、邮件、群内公告）告知当事人。

### 5.7 申诉机制

当事人对处罚决定有异议的，可在收到通知后7日内向管理组提交书面申诉。管理组应在收到申诉后15日内作出复核决定。复核期间不中止原处罚的执行。

### 5.8 处罚记录与公示

管理组有权在不泄露个人隐私的前提下，对严重违规及最终处罚结果进行适当公示（如隐去部分个人信息），以警示其他成员。
